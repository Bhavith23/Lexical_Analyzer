# Lexical_Analyzer
Developed a Lexical Analyzer in C that scans source code and identifies different lexical tokens used in programming languages. The analyzer processes input files character by character and classifies lexemes into categories such as keywords, identifiers, operators, constants, special symbols, and literals.

Key Features:
- Identification of keywords and identifiers
- Recognition of operators and delimiters
- Detection of numeric constants and literals
- Token classification and reporting
- File-based source code processing
- Error detection for invalid tokens
- Efficient character-by-character scanning
- Modular implementation for easy maintenance

Technologies Used:
- C Programming
- File Handling
- String Manipulation
- Data Structures
- Compiler Design Concepts

Learning Outcomes:
- Understanding of compiler design fundamentals
- Lexical analysis and tokenization techniques
- Pattern matching and string processing
- File parsing and source code analysis
- Development of modular and efficient C programs

This project demonstrates the first phase of compiler design by converting source code into meaningful tokens, which can be further utilized by syntax and semantic analysis stages.
